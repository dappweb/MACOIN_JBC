# 奖励策略对比报告 (Reward Strategy Comparison)

本报告详细对比了当前协议中的静态收益（Static Rewards）与极差奖励（Differential Rewards/Team Rewards）策略。

## 1. 静态收益 (Static Rewards)

静态收益是用户通过质押流动性（MC/JBC）获得的固定回报。收益率根据质押周期不同而变化。

### 收益率设定
| 质押周期 | 日收益率 (Daily Rate) | 备注 |
| :--- | :--- | :--- |
| **7 天** | **1.33%** | 短期灵活，收益较低 |
| **15 天** | **1.66%** | 中期平衡 |
| **30 天** | **2.00%** | 长期质押，收益最高 |

### 计算逻辑
- **基数**: 质押的流动性价值 (MC等值)
- **结算**: 按天结算 (Seconds in Unit)
- **公式**: `收益 = 质押金额 * 日收益率 * 质押天数`
- **发放**: 赎回时一次性发放，50% MC + 50% JBC (自动回购)

---

## 2. 极差奖励 (Differential Rewards / Team Rewards)

极差奖励是基于团队规模（有效地址数）的级差奖励机制。当团队成员获得静态收益时，上级根据等级差获得奖励。

### 等级标准 (V1-V9)
等级由**团队总人数 (Team Count)** 决定：

| 等级 (Level) | 团队人数要求 (Team Count) | 极差比例 (Percent) |
| :--- | :--- | :--- |
| **V1** | ≥ 10 | 5% |
| **V2** | ≥ 30 | 10% |
| **V3** | ≥ 100 | 15% |
| **V4** | ≥ 300 | 20% |
| **V5** | ≥ 1,000 | 25% |
| **V6** | ≥ 3,000 | 30% |
| **V7** | ≥ 10,000 | 35% |
| **V8** | ≥ 30,000 | 40% |
| **V9** | ≥ 100,000 | 45% |

### 发放逻辑
- **触发**: 下级赎回质押并获得静态收益时触发。
- **计算**: `奖励 = 下级静态收益 * (上级比例 - 下级最高比例)`
- **机制**: 极差制 (Differential)。例如，V3 (15%) 的下级是 V1 (5%)，则 V3 获得 `15% - 5% = 10%` 的差额奖励。
- **烧伤**: 如果上级比例 <= 下级比例，则不获得奖励（平级或被超越）。

---

## 3. 策略核心差异

| 特性 | 静态收益 | 极差奖励 |
| :--- | :--- | :--- |
| **收益来源** | 个人资金质押 | 团队裂变与管理 |
| **风险/成本** | 资金占用风险 | 推广时间与精力成本 |
| **上限** | 受限于个人本金与当前额度 (Current Cap) | 受限于团队总业绩与烧伤机制 |
| **发放代币** | MC + JBC | MC (记录为待领取) |
| **被动性** | 完全被动 | 需要持续维护团队活跃度 |
