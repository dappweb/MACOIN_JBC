# Test vs P-prod 商业模式与代币模式差异分析需求文档

## 介绍

本文档分析Jinbao Protocol在test分支和p-prod分支之间的商业模式和代币经济模型差异，为用户和开发者提供清晰的环境对比信息。

## 术语表

- **Test环境**: 测试分支环境，用于功能验证和开发测试
- **P-prod环境**: 生产预发布分支环境，接近真实生产环境
- **商业模式**: 协议的收入来源、分配机制和经济激励结构
- **代币模式**: MC和JBC代币的发行、分配、燃烧和交易机制
- **时间单位**: SECONDS_IN_UNIT，影响所有时间相关计算的基础参数
- **奖励分配**: 直推、层级、极差等各类奖励的百分比配置

## 需求

### 需求1: 商业模式差异分析

**用户故事**: 作为协议用户，我想了解test和p-prod环境的商业模式差异，以便选择合适的环境进行参与。

#### 验收标准

1. WHEN 用户查询商业模式信息 THEN 系统应显示两个环境的收入来源对比
2. WHEN 用户查询奖励机制 THEN 系统应显示奖励分配百分比的差异
3. WHEN 用户查询时间机制 THEN 系统应显示质押周期和燃烧周期的实际差异
4. WHEN 用户查询钱包配置 THEN 系统应显示资金流向和管理方式的差异
5. WHEN 用户查询业务活跃度 THEN 系统应显示交易量和用户参与度的对比

### 需求2: 代币经济模型对比

**用户故事**: 作为投资者，我想了解两个环境的代币经济模型差异，以便评估投资风险和收益。

#### 验收标准

1. WHEN 用户查询代币供应 THEN 系统应显示MC和JBC的供应量差异
2. WHEN 用户查询燃烧机制 THEN 系统应显示代币燃烧频率和数量的差异
3. WHEN 用户查询流动性 THEN 系统应显示AMM池储备和价格机制的差异
4. WHEN 用户查询交易税费 THEN 系统应显示买卖税费配置的对比
5. WHEN 用户查询价格稳定性 THEN 系统应显示代币价格波动和稳定机制的差异

### 需求3: 时间机制影响分析

**用户故事**: 作为质押用户，我想了解时间单位配置对实际业务的影响，以便做出正确的参与决策。

#### 验收标准

1. WHEN 用户查询质押周期 THEN 系统应显示7/15/30天质押的实际持续时间
2. WHEN 用户查询奖励发放 THEN 系统应显示奖励解锁时间的实际差异
3. WHEN 用户查询燃烧周期 THEN 系统应显示日燃烧机制的实际执行频率
4. WHEN 用户查询门票有效期 THEN 系统应显示72小时灵活期的实际时长
5. WHEN 用户查询极差奖励 THEN 系统应显示30天解锁期的实际等待时间

### 需求4: 风险与机会评估

**用户故事**: 作为协议参与者，我想了解两个环境的风险和机会差异，以便制定合适的参与策略。

#### 验收标准

1. WHEN 用户查询投资风险 THEN 系统应显示配置异常和潜在风险的对比
2. WHEN 用户查询收益机会 THEN 系统应显示奖励机制和收益潜力的差异
3. WHEN 用户查询流动性风险 THEN 系统应显示资金安全和提取便利性的对比
4. WHEN 用户查询技术风险 THEN 系统应显示合约稳定性和升级风险的差异
5. WHEN 用户查询市场风险 THEN 系统应显示代币价格波动和市场深度的对比

### 需求5: 环境选择建议

**用户故事**: 作为新用户，我想获得环境选择建议，以便选择最适合我需求的参与环境。

#### 验收标准

1. WHEN 新用户查询环境选择 THEN 系统应提供基于用户类型的环境推荐
2. WHEN 用户查询测试需求 THEN 系统应推荐test环境并说明适用场景
3. WHEN 用户查询生产参与 THEN 系统应推荐p-prod环境并说明注意事项
4. WHEN 用户查询风险承受 THEN 系统应基于风险偏好提供环境建议
5. WHEN 用户查询功能需求 THEN 系统应基于功能差异提供选择指导

### 需求6: 实时监控与预警

**用户故事**: 作为系统管理员，我想监控两个环境的关键差异变化，以便及时发现和处理异常。

#### 验收标准

1. WHEN 系统检测到配置差异 THEN 应自动生成差异报告并发送预警
2. WHEN 系统检测到时间单位异常 THEN 应立即发出高危预警
3. WHEN 系统检测到奖励分配变化 THEN 应记录变更历史并通知相关人员
4. WHEN 系统检测到代币供应异常 THEN 应触发安全检查流程
5. WHEN 系统检测到业务数据异常 THEN 应启动数据验证和修复流程

### 需求7: 迁移与同步指导

**用户故事**: 作为开发者，我想了解如何在两个环境之间进行配置同步和数据迁移，以便维护系统一致性。

#### 验收标准

1. WHEN 开发者需要同步配置 THEN 系统应提供配置同步的详细步骤
2. WHEN 开发者需要迁移数据 THEN 系统应提供数据迁移的安全方案
3. WHEN 开发者需要修复差异 THEN 系统应提供差异修复的操作指南
4. WHEN 开发者需要验证一致性 THEN 系统应提供自动化验证工具
5. WHEN 开发者需要回滚变更 THEN 系统应提供安全回滚的操作流程
