# Jinbao Protocol 正确的商业模式详细说明

## 🎯 四种奖励机制详解

### 1️⃣ 静态奖励 (质押挖矿收益)

**定义**: 用户质押门票获得的固定日收益

**运作机制**:
```
质押周期和收益率:
├── 7天质押: 2.0%日收益 (真实7天)
├── 15天质押: 2.5%日收益 (真实15天)
└── 30天质押: 3.0%日收益 (真实30天)

计算公式:
日收益 = 质押金额 × 日收益率
总收益 = 日收益 × 质押天数
```

**实例计算**:
- 质押1000 MC，选择30天周期
- 日收益 = 1000 × 3.0% = 30 MC
- 总收益 = 30 × 30 = 900 MC
- 年化收益率 = 3.0% × 365 = 1095%

### 2️⃣ 动态奖励 (推荐奖励)

**定义**: 基于推荐网络产生的奖励，包含直推奖励

**运作机制**:
```
直推奖励:
├── 奖励比例: 25% MC
├── 解锁时间: 即时解锁
├── 奖励来源: 直接推荐用户的门票购买金额
└── 计算方式: 被推荐人门票金额 × 25%

实例:
- A推荐B购买1000 MC门票
- A获得直推奖励: 1000 × 25% = 250 MC (即时到账)
```

### 3️⃣ 层级奖励 (多层推荐奖励)

**定义**: 基于推荐网络深度的多层级奖励

**运作机制**:
```
层级奖励结构:
├── 奖励层数: 15层
├── 每层奖励: 1% MC
├── 解锁时间: 即时解锁
└── 累计奖励: 最高15% (1% × 15层)

计算逻辑:
第1层 (直推的推荐): 门票金额 × 1%
第2层 (直推的推荐的推荐): 门票金额 × 1%
...
第15层: 门票金额 × 1%
```

**实例**:
```
推荐网络: A → B → C → D → E → ...
E购买1000 MC门票时:
├── D获得第1层奖励: 1000 × 1% = 10 MC
├── C获得第2层奖励: 1000 × 1% = 10 MC
├── B获得第3层奖励: 1000 × 1% = 10 MC
├── A获得第4层奖励: 1000 × 1% = 10 MC
└── 以此类推到第15层
```

### 4️⃣ 级差奖励 (V等级极差奖励)

**定义**: 基于V等级差额的高级奖励机制

**V等级系统**:
```
V等级收益比例:
├── V0: 0人团队, 0%极差收益
├── V1: 10+人团队, 5%极差收益
├── V2: 30+人团队, 10%极差收益
├── V3: 100+人团队, 15%极差收益
├── V4: 300+人团队, 20%极差收益
├── V5: 1000+人团队, 25%极差收益
├── V6: 3000+人团队, 30%极差收益
├── V7: 10000+人团队, 35%极差收益
├── V8: 30000+人团队, 40%极差收益
└── V9: 100000+人团队, 45%极差收益
```

**级差奖励计算**:
```
极差奖励 = 门票金额 × (自己V等级% - 上级V等级%)
解锁时间: 30天 (真实30天)
奖励条件: 只有当自己V等级高于上级时才获得极差
```

**实例**:
```
推荐链: A(V5-25%) → B(V3-15%) → C(V1-5%) → D(V0-0%)
D购买1000 MC门票时:

C获得极差: 1000 × (5% - 0%) = 50 MC (30天后解锁)
B获得极差: 1000 × (15% - 5%) = 100 MC (30天后解锁)  
A获得极差: 1000 × (25% - 15%) = 100 MC (30天后解锁)
```

## 🔥 燃烧机制 (不分红给用户)

**重要说明**: 燃烧机制是通缩机制，不直接分红给用户

**燃烧机制**:
```
燃烧目的:
├── 减少JBC总供应量
├── 创造通缩价值
├── 提升剩余JBC价值
└── 不分配给用户，纯销毁

燃烧参数:
├── 燃烧周期: 每24小时 (真实24小时)
├── 燃烧比例: 合约JBC余额的1%
├── 燃烧方式: 转移到黑洞地址永久销毁
└── 燃烧效果: 总供应量持续减少
```

## 💎 双代币模型

### MC代币 (Master Coin)
```
用途:
├── 门票购买: 100/300/500/1000 MC
├── 质押本金: 获得静态奖励
├── 奖励支付: 动态、层级、级差奖励都用MC支付
└── 流动性要求: 购买门票需要对应MC流动性
```

### JBC代币 (Jinbao Coin)
```
用途:
├── 燃烧销毁: 每日燃烧减少供应量
├── 通缩价值: 通过销毁提升价值
├── 内置闪兑: 与MC互换 (25%/50%销毁)
└── 价值存储: 受益于通缩机制
```
